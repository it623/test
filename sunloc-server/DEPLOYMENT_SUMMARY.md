## 🚀 Fastest Path to Production

### Option 1: Using Railway CLI (5 minutes)

```bash
# 1. Install Railway CLI
npm install -g @railway/cli

# 2. Login
railway login

# 3. Initialize project
railway init

# 4. Create PostgreSQL plugin
railway add postgres

# 5. Set environment variables
railway variables set NODE_ENV=production PORT=3000

# 6. Deploy
railway up
```

**✅ Done!** Your server is live. Get URL with:
```bash
railway open
```

### Option 2: Via GitHub (10 minutes)

```bash
# 1. Push to GitHub
git add .
git commit -m "Sunloc Server PostgreSQL"
git push

# 2. Go to https://railway.app
# 3. New Project → Deploy from GitHub
# 4. Select repo → Railway auto-deploys
# 5. Add PostgreSQL plugin
# 6. Set env vars from PostgreSQL credentials
# 7. Check logs for success
```

## 📋 Railway Environment Variables

Copy these from your PostgreSQL plugin and set in Railway dashboard:

```
DB_HOST=postgres.railway.internal
DB_PORT=5432
DB_NAME=railway
DB_USER=postgres
DB_PASSWORD=<generated by Railway>
PORT=3000
NODE_ENV=production
```

## ❓ Troubleshooting

**Server won't start:**
- Check Railway Logs tab
- Verify all DB_* variables are set
- PostgreSQL might need 30s to initialize

**Connection error:**
- Use `postgres.railway.internal` (not `localhost`)
- Ensure PostgreSQL plugin is in same project

**Data not syncing:**
- Check `/api/health` returns 200 OK
- Verify app URLs point to Railway domain
- Check browser console for fetch errors
